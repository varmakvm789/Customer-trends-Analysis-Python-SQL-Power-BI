## 📌 Overview

This project represents a **complete, industry-standard, end-to-end data analytics workflow**, designed to mirror the real responsibilities of professional analysts in modern business environments. Using a retail customer shopping behavior dataset, this project walks through every critical stage — from raw data to business-ready insights and presentations.

This project is perfect for:
- 📊 Data Analyst aspirants who want to build a strong **Portfolio Project** for interviews and LinkedIn
- 📚 Anyone learning Python, SQL, and Power BI
- 💼 Professionals preparing for interviews in **Data Analytics**, **Data Science**, or **Product Analytics** roles

---

## 🗂️ Dataset

- **Source:** Retail Customer Shopping Behavior Dataset
- **Format:** CSV
- **Key Fields:** Customer ID, Age, Gender, Purchase Amount, Product Category, Payment Method, Review Rating, Subscription Status, Frequency of Purchases, Season, Discount Applied, Promo Code Used

## 📋 Project Steps

### Step 1 — Business Problem Statement
Define the core business questions the analysis will address, including customer segmentation, loyalty patterns, purchase drivers, and seasonal trends.

### Step 2 — Data Modelling & EDA in Python
- Import the raw CSV dataset using **Pandas**
- Explore data shape, types, null values, and distributions
- Clean the data: handle missing values, fix dtypes, remove duplicates
- Perform EDA with **Matplotlib** and **Seaborn**: distributions, correlations, category breakdowns

### Step 3 — Data Analysis in SQL (PostgreSQL)
- Load the cleaned dataset into a **PostgreSQL** database via Python (`SQLAlchemy`)
- Write SQL queries to answer business questions:
  - Which customer segments generate the highest revenue?
  - What drives loyalty and repeat purchases?
  - How does purchase behavior vary by season, gender, or location?

### Step 4 — Interactive Dashboard in Power BI
- Connect Power BI directly to the PostgreSQL database
- Build interactive visuals: KPI cards, bar charts, pie charts, slicers, maps
- Enable stakeholders to drill down by category, region, season, and payment method

### Step 5 — Project Report
- Summarize key findings, methodology, and limitations
- Translate analytical results into plain-language business recommendations

### Step 6 — Presentation using Gamma AI
- Build a visually polished stakeholder presentation using **Gamma AI**
- Communicate findings and recommendations clearly and concisely

### Step 7 — GitHub Repository
- Organize and upload all project files
- Write a professional README for visibility and portfolio impact

## ▶️ How to Run

### Prerequisites
- Python 3.8+
- PostgreSQL (or MySQL / MS SQL Server)
- Power BI Desktop
- Required Python libraries: `pandas`, `numpy`, `matplotlib`, `seaborn`, `sqlalchemy`, `psycopg2`

### Steps

**1. Install dependencies**
```bash
pip install pandas numpy matplotlib seaborn sqlalchemy psycopg2-binary
```

**2. Open the Python Notebook**
```
Customer_Shopping_Behavior_Analysis.ipynb
```
- Import and explore the dataset
- Clean and transform the data
- Load the cleaned data into your PostgreSQL database

**3. Run SQL Queries**
```
customer_behavior_sql_queries.sql
```
- Open in your SQL client (DBeaver, pgAdmin, etc.)
- Run queries to extract business insights

**4. Open Power BI Dashboard**
```
customer_behavior_dashboard.pbix
```
- Connect to your PostgreSQL database
- Refresh data and explore the interactive dashboard

**5. View the Report & Presentation**
- Project report summarizes findings and recommendations
- Presentation built with Gamma AI for stakeholder communication
